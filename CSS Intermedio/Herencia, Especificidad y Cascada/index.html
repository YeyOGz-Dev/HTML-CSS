Herencia, Especificidad y Cascada en CSS

Herencia

La herencia es un concepto clave que determina cómo las propiedades de estilo de los elementos HTML se heredan a sus elementos hijos. Hay propiedades que si se heredan y otras no:

1. Propiedades que se heredan por defecto:

  • color
  • font-family
  • font-size
  • line-heigth
  • text-align

2. Propiedades que NO se heredan

  • margin
  • padding
  • border
  • background
  • width

3. Propiedad "inherit"

Esta propiedad sirve para forzar un valor que no es heredable

4. Valores "initial" y "unset"

  • initial: Restablece la propiedad al valor predeterminado del navegador.
  • unset: Establece la propiedad al valor predeterminado si no es heredable o la hereda si es heredable.

Especificidad

La especificidad define qué tan especifica o fuerte es una regla en CSS para un elemento en particular. Cuanto mas específica sea una regla, mayor será su prioridad. Los selectores de especificidad se dividen en cuatro categorias:

  1. Estilos en linea (atributo style en HTML)
  2. Selectores de ID ( #mi-id )
  3. Selectores de Clase ( .mi-clase ), pseudoclases ( :hover, :nth-child ) y selectores de atributos ( [type="text"] )
  4. Selectores de tipo o etiqueta ( div, p, h1 ) y pseudoelementos ( ::before, ::after )

Valor de cada selector:

  • Estilo en Linea (1,0,0,0)
  • Selector de ID: (0,1,0,0)
  • Selector de clase: (0,0,1,0)
  • Selector de tipo o etiqueta: (0,0,0,1)

Reglas importantes sobre la especificidad

  • Los selectores de ID son más específicos que los selectores de clase.
  • Los selectores de clase son más específicos que los selectores de tipo o etiqueta.
  • Si varios selectores tienen la misma especificidad, se aplica el que aparece más tarde en el CSS.
  • El valor !important puede anular la especificidad, pero debe usarse con precaución. en el mejor de los casos es mejor no usar esta regla.

Cascada

La cascada en CSS es el proceso mediante el cual el navegador decide cuál de lasl reglas aplicadas a un mismo elemento debe tener prioridad. Se basa en tres principios:

Origen de las reglas

  • Hojas de estilo del navegador (Por defecto)
  • Hojas de estilo del autor (Las que se desarrollan)
  • Hojas de estilo del usuario (Si el usuario ha personalizado su navegador)

Especificidad

Los estilos con mayor especificidad tienen prioridad sobre los menos especificos.

Orden de aparición

Si dos reglas tienen la misma especificidad, la última en aparecer en el codigo tendrá prioridad.

La propiedad " !IMPORTANT "

Usar esta propiedad en una declaración de estilo hace que esta regla tenga la máxima prioridad. Independientemente de la especificidad o el orden.

En conclusion, Lo que es la Especificidad y la Cascada en CSS trabajan juntos para definir cómo se aplican los estilos en una página.
