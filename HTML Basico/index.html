¿Que es HTML?

Hypertext Markup Language (Lenguaje de marcado de hipertexto) Estructura (Esqueleto) de toda la pagina web.

¿Que es el SEO?

El SEO (Search engine optimization - Optimizacion para motores de busqueda) Es el conjunto de tecnicas y estrategias que se aplican a una página web para mejorar su posicionamiento organico en los resultados de busqueda en los navegadores. El objetivo principal del SEO es lograr que la página web aparezca entre los primeros resultados cuando los usuarios realicen busquedas relacionadas con la tematica de la pagina web. En conclusion , el SEO es una estrategia fundamental para cualquier pagina que quiera darse a conocer.

¿Que es HTTPS?

HyperText Transfer Protocol Secure (Protocolo seguro de transferencia de hipertexto)

Hay 6 tipos de encabezados, desde el h1 al h6. Solo se debe usar un 'h1' por pagina ya que esto ayuda al SEO. de resto se pueden poner de la h2 a h6 las que sean necesarias.

Enlaces (a href)

La etiqueta 'a' viene con el atributo 'href', este es para darle el valor de la url donde queremos ir, ya sea interna o externa. Para los enlaces esta el relativo y el absoluto, el enlace relativo es una ruta que se define en relacion a la ubicacion actual del archivo HTML por ejemplo: /index.html. El enlace absoluto es una ruta completa que incluye el protocolo, el nombre del dominio y la ruta del archivo por ejemplo: https://Youtube.com/YeisonGil

    Formularios (form)

    Los formularios son elementos que permiten que los usuarios interactuen con la pagina web y enviar datos al servidor. estos se usan para recopilar datos de un usuario, permitir registros en el sitio web, realizar pedidos en linea, enviar comentarios o sugerencias etc.

    Atributos importantes para los inputs:

      Atributos basicos
    • type: Este es para darle el tipo de input, ya sea de tipo texto o email etc.
    • name: Asigna un nombre único al campo de entrada. Es necesario para poder identificar y recuperar el valor ingresado por el usuario posteriormente. Esto es para cuando los datos se envien al servidor.
    • value: Este es para darle un 'nombre descritivo al input'
      • Atributos de control
    • checked: Este solo funciona para los inputs de tipo checkbox y radio. Sirve para mostrar si el item esta seleccionado, por defecto es no.
    • readonly: Deshabilita la edición del campo, quedando como texto informativo.
    • disabled: Deshabilita el campo por completo, sin permitir interacción del usuario.
    • required: Este es para que el campo sea obligatorio
      • Atributos de tamaño y presentacion
    • size: Define el ancho del campo de entrada en caracteres (aproximadamente).
    • maxlength: Limita el número máximo de caracteres permitidos en el campo.
    • minlength: Limita el número minimo de caracteres permitidos en el campo.
    • placeholder: Este es para mostrar al usuario un ejemplo de lo que debe escribir en el campo y desaparece cuando el usuario empieza a escribir.
      • Otros atributos
    • autocomplete: Activa o desactiva el autocompletado del navegador según el tipo de campo.
    • pattern: Define una expresión regular para validar el formato de los datos introducidos.
    • id: Asigna un identificador único al elemento input, permitiendo un mejor control con CSS o JavaScript.
